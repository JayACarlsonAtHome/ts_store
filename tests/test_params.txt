# ts_store test parameters
#
# DISK_TYPE: x7k (7200rpm), 10k (10000rpm), ssd -- all exactly 3 chars
#   so that disk subdirs line up vertically when ls'ing: x7k/ 10k/ ssd/
#   (CLI --disk accepts 7k/7200/x7k etc. and normalizes)
#
# SIZE: smoke (tiny, ~100 events, safe) or full (xFull matrix, ~30 min on x7k).
#
# In SIZE=full the runner uses *progressive* difficulty so that:
#   001     : hello-world sanity (512 records)
#   002     : small (~768 records)
#   003-004 : medium (~1.5k–2.5k records)
#   005/007 : heavy (50 threads × 2000 events × 3 runs = 300k events; 100k records in manifest)
#   006     : tail-reader stress (50 × 2000 = 100k events, single pass)
#
# This prevents "test 001 taking an hour" when you want full stress on the big tests.
# The old blunt global override has been replaced by per-test scaling in the runner.
#
# OS_ID is set automatically by ./scripts/Build from FileCheckList.txt section
# headers (#OS_001, #OS_002, #OS_003, ...). Each [x] row inherits the current
# section's OS_ID for test-results/OS_00n/... and test-summary/OS_00n/...
#
# Optional override here (or ts_test_cli --os-id) for manual runs outside Build.
#
# The OS_MAP.txt will contain the human list:
#   OS_001 = RHEL 9.8
#   OS_002 = RHEL 10.2
#   OS_003 = Linux Mint
#
# Result directories stay short for alignment (padded OS_00n + compiler leaf):
#   test-results/OS_001/gcc/x7k/xFull/...
#   test-results/OS_001/clang/10k/Smoke/...
#
# Each test-results leaf also gets OS_INFO.txt + manifest HostInfo (CPU cores,
# RAM, model, hostname) captured at run start — promoted to test-summary/.
#
# SIZE_LABEL is auto-derived:
#   SIZE=smoke  -> Smoke
#   SIZE=full   -> xFull
#
# If no OS_ID (auto or override), the older flat layout test-results/<DISK_TYPE>/ is used.
#
# Number formatting in CLI summaries and benchmark stats (ops/sec, µs, events):
#   Default: comma grouping (10,564,124) unless locale env vars are set
#   TS_STORE_NUMBER_LOCALE=en_US.utf8  — force a locale name
#   TS_STORE_THOUSANDS_SEP=.           — explicit separator (comma or dot)
#   LC_NUMERIC / LANG                  — use system locale grouping
